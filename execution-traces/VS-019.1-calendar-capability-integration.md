# VS-019.1 — Integração da Capability Calendar

## Âmbito

`CALENDAR_CREATE_EVENT` é uma capability externa distinta. O Kernel reconhece um pedido de criação de reunião, cria uma `CapabilityRequest`, avalia a policy, materializa um `CalendarEventDraft`, prepara a execução e persiste uma `ApprovalRequest` antes de tocar na API Google.

```text
Mensagem estruturada
        ↓
CapabilityRequest(CALENDAR_CREATE_EVENT)
        ↓
Policy(REQUIRED)
        ↓
CalendarEventDraft
        ↓
ExecutionPreparation(PENDING_APPROVAL)
        ↓
ApprovalRequest(PENDING)
        ↓
Approval(APPROVED)
        ↓
GoogleCalendarProvider.create_event
        ↓
CapabilityResult(SUCCESS | FAILED)
```

## Contrato de entrada inicial

Para evitar ambiguidade temporal nesta fase, o pedido usa datas ISO 8601 explícitas:

```text
Marca uma reunião | Título: Revisão | Início: 2026-07-20T15:00:00+01:00 | Fim: 2026-07-20T16:00:00+01:00 | Convidados: joao@example.com
```

`Título`, `Início` e `Fim` são obrigatórios; `Convidados` e `Fuso` são opcionais. Uma preparação inválida é bloqueada antes de pedir aprovação.

## Garantias

- Nenhum evento é criado antes de `Approval(APPROVED)`.
- A `ExecutionRecord` é persistida antes da chamada externa.
- A mesma execução não é repetida automaticamente após falha ou reinício.
- A chave de idempotência da Aurora é enviada como ID do evento Google e como propriedade privada do evento.
- A consulta de disponibilidade (`freeBusy`) permanece um incremento futuro: ainda não existe uma capability de leitura ligada ao Kernel.
